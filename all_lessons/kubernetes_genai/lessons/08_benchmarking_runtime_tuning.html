Linear Story

Production tuning starts with a workload contract. A benchmark without prompt lengths, output lengths, concurrency, sampling settings, model version, hardware, and routing policy is mostly a story about the benchmark machine. The book moves from evaluation to runtime parameters because the platform must optimize both quality and throughput.

LLM serving has two main timing surfaces. TTFT is dominated by queueing, routing, model readiness, and prefill. TPOT is dominated by decode scheduling, memory bandwidth, KV cache pressure, and batching. A change that improves throughput can make TTFT worse. A change that improves p50 can destroy p99. A quantization change can lower cost and silently degrade the task.

Runtime Knobs as a Coupled System

vLLM-style knobs such as maximum batched tokens, maximum sequences, GPU memory utilization, KV cache dtype, prefix caching, chunked prefill, speculative decoding, and quantization are coupled. Raising concurrency increases utilization until KV cache pressure causes queueing or preemption. Chunked prefill can mix long prompts with decode work, but it changes latency shape. Prefix caching is powerful only when routing and prompt structure create reuse.

KnobImprovesCan hurt
Higher batch/concurrencyThroughput and GPU utilizationQueueing and p99 latency
QuantizationMemory, cost, sometimes speedQuality and runtime compatibility
Prefix cachingRepeated prompt prefill costMemory pressure if reuse is low
Chunked prefillMixed workload utilizationMore complex latency tuning

Evaluation Pairing

Every performance experiment should carry a quality check. Use task evals for correctness, style, safety, and regression. The platform should be able to say: this config is cheaper by 20%, but it changes answer quality by this measured amount on these tasks. That sentence is what turns tuning from folklore into engineering.

Mental Model

A benchmark is a workload contract. Without prompt length, output length, concurrency, sampling settings, and hardware topology, throughput numbers are mostly decoration.

Operational Checklist

  • Benchmark prefill-heavy and decode-heavy workloads separately.
  • Track TTFT, TPOT, tokens per second, GPU utilization, HBM pressure, queue time, and quality metrics together.
  • Tune max batched tokens, max sequences, KV cache dtype, quantization, and prefix caching as a coupled set.
  • Compare quality before and after compression or quantization, not only speed.

Failure Modes

  • A synthetic short-prompt benchmark predicts production badly because real prompts are long and cacheable.
  • Quantization improves cost but silently degrades the task that matters.
  • Runtime knobs are tuned for peak throughput and blow up p99 latency.
