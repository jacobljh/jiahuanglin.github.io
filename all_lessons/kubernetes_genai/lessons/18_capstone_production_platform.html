Linear Story

The capstone is the whole platform loop. A model is proposed, packaged, evaluated, promoted, deployed, routed, observed, protected, tuned, and sometimes rolled back. Incidents and eval failures feed back into tests and policy. This lesson adds the production layer that the book implies across chapters: release management, governance, FinOps, and incident response.

Release Management

Use GitOps for model references, runtime settings, routing rules, and platform configuration. Gate changes with offline evals, compatibility checks, license checks, security scans, and cost estimates. Promote with shadow traffic or canaries. Define rollback before rollout. A model release without a rollback plan is not a release; it is a bet.

Governance and Security

The security boundary covers more than pods. It includes model artifacts, registries, prompts, retrieved documents, tool calls, logs, adapters, training data, and human approval flows. Use RBAC, NetworkPolicy, Pod Security, secrets management, artifact signing, SBOMs, license checks, PII redaction, and tenant isolation as layers. The platform should make the secure path the easy path.

FinOps

GPU cost must be visible in workload terms: dollars per GPU-hour, dollars per million input tokens, dollars per million output tokens, idle GPU cost, cold-start waste, cache hit savings, and cost per successful task. Quota, admission control, and load shedding are product controls, not only finance controls.

Final Mental Model

Production GenAI on Kubernetes is not "run vLLM in a pod." It is a controlled system for moving models, tokens, identity, state, evidence, and money through a cluster with measurable quality and reversible change.

Mental Model

A production platform is a control loop: propose a model/config change, evaluate it, promote it, watch it, roll it back if needed, and feed incidents back into tests and policy.

Operational Checklist

  • Use GitOps for runtime config, routing policy, model references, and infrastructure changes.
  • Gate promotion with offline evals, shadow traffic, canary metrics, and rollback criteria.
  • Track GPU capacity, model cost, dollars per 1M tokens, quota, admission control, and load shedding.
  • Secure the full path: RBAC, NetworkPolicy, Pod Security, secrets, signing, SBOMs, license checks, and tenant isolation.

Failure Modes

  • A new model is deployed without an eval baseline or fast rollback path.
  • Cost incidents are treated as finance surprises instead of platform SLO failures.
  • Security controls cover the app but not model artifacts, prompts, retrieved data, or tool calls.
