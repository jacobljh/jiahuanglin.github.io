Linear Story

After deployment comes the thing being deployed: model data. The book treats model artifacts as a first-class operational problem because LLM weights are too large, too regulated, and too coupled to runtime assumptions to be treated like incidental files. A model version includes more than parameters: tokenizer, config, license, quantization, base model lineage, adapter relation, intended runtime, and evaluation record.

The storage format answers "can this runtime load it efficiently?" Safetensors is common for transformer checkpoints because it is safe to load and friendly to modern serving stacks. GGUF/GGML matter for llama.cpp-style local and CPU/GPU hybrid deployments. ONNX matters when the target runtime is an ONNX-serving ecosystem or when graph portability is the goal. The registry answers a different question: "which artifact is approved, discoverable, and promotable?"

Registry as Control Plane

Hugging Face is a discovery and distribution hub. MLflow Model Registry gives lifecycle states and experiment lineage. Kubeflow Model Registry fits a Kubernetes-centric ML platform. OCI registries turn models into content-addressed artifacts with familiar promotion, caching, signing, and access-control semantics. The best choice depends less on taste and more on which system your release process already trusts.

Registry pathStrengthRisk
Hugging FaceBroad ecosystem and model discoveryExternal availability, licensing, and promotion control

Trade-Off

The central trade-off is velocity versus governance. Pulling a model directly from an open hub is fast and useful during exploration. Promoting a signed, versioned OCI artifact through dev, staging, and prod is slower but gives reproducibility. Production usually needs the latter, even if research starts with the former.

Thinking Drill

Before deploying any model, ask: Can I name the exact artifact digest? Can I reconstruct which tokenizer was used? Can I explain the license? Can I roll back without downloading a mutable tag? Can I correlate production traffic with an evaluation report? If any answer is no, the model is not yet a production artifact.

Mental Model

The registry is the source of truth for promotion. The format is the compatibility contract with the runtime. The cluster mount path is only the last mile.

Operational Checklist

  • Track model identity, version, license, base model, quantization, tokenizer, and runtime compatibility.
  • Prefer safetensors for common transformer serving unless the runtime needs another format.
  • Use OCI registries when artifact distribution, caching, signing, and promotion should mirror container workflows.
  • Make model metadata queryable by humans and automation before it reaches production.

Failure Modes

  • A model version changes while the Deployment image tag remains the same.
  • The tokenizer and weights are promoted separately and drift.
  • Licensing or export constraints are discovered after the model is already cached on production nodes.
