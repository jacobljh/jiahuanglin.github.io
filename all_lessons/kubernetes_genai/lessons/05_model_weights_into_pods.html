Linear Story

Once the model is versioned, the platform must put it where the runtime can read it. The book covers PersistentVolumes, sidecar-style modelcars, and OCI image volume ideas because the delivery path controls cold start. A model server that starts instantly in a local demo may spend most of its production startup pulling and reading weights.

registry/object store/PV
        |
        v
node cache or shared storage
        |
        v
pod mount path -> runtime loads weights -> readiness true

The key distinction is between distributing the artifact and reading the artifact. An OCI registry can distribute immutable bytes and benefit from node cache semantics. A PV can expose shared bytes to many pods. An init container can copy bytes into an ephemeral volume. Each path changes who pays for bandwidth, how rollback works, and whether scale-up is predictable.

Cold-Start Equation

Think of startup as: image pull + model fetch + model read + runtime initialization + warmup. Only the last two are model-server concerns. The first three are platform concerns. If a scale-from-zero path has to pull a 100 GB model over the network before readiness, autoscaling is no longer just scheduling more pods; it is staging huge artifacts under time pressure.

Delivery pathGood fitTrade-off
PersistentVolumeShared model data and stable clustersStorage throughput and mount semantics become critical
Init copySimple workflows and local ephemeral speedRepeated copy cost at every restart
ModelcarOCI-style packaging before native image volumesSidecar lifecycle and multi-arch complexity
Image volumeImmutable OCI artifact mounted read-onlyRuntime support and feature maturity must be verified

Operational Rule

Measure model load as its own metric. Do not hide it inside pod startup. A healthy platform can answer: was this pod slow because the node lacked the model layer, because storage was slow, because CUDA initialization was slow, or because the runtime was compiling/warming kernels?

Mental Model

A 100 GB model is not configuration. It is a deployable supply-chain artifact whose delivery path determines startup time, node pressure, rollback speed, and cache hit rate.

Operational Checklist

  • Use PVs when many pods need shared read access and the storage backend has predictable throughput.
  • Use modelcars or OCI image volumes when registry distribution, immutability, and node cache semantics are valuable.
  • Pre-pull or prefetch large models before scale-up if cold-start SLOs matter.
  • Measure node-local cache hit rate and model load time separately from HTTP process startup.

Failure Modes

  • The deployment scales from zero but spends minutes pulling weights before readiness.
  • Every replica downloads the same model from a remote source and saturates egress or object storage.
  • Mutable tags make rollback non-reproducible.
