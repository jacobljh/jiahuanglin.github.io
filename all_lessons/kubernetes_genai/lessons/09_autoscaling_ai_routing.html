Linear Story

Generic autoscaling says: more requests, add pods. GenAI autoscaling says: more tokens, longer prompts, deeper queues, hotter adapters, colder caches, or saturated GPUs may require different actions. The book's routing chapter is important because the router is no longer a blind load balancer. It is a model-aware scheduler above Kubernetes scheduling.

A router can preserve prefix-cache locality, send LoRA requests to replicas where the adapter is already loaded, enforce tenant policy, shape traffic during rollout, and expose token-aware metrics. The cost is that routing becomes stateful and policy-heavy. Fairness, locality, and latency can conflict.

Scaling Signals

CPU is often a weak signal for LLM serving. Better signals include request queue time, active sequences, prompt tokens per second, output tokens per second, TTFT, TPOT, GPU utilization, HBM/KV pressure, and model load state. Scale-up must consider cold start: a new replica that needs minutes to fetch weights cannot immediately save a latency incident.

Gateway Boundary

An AI gateway is a useful boundary for authentication, rate limits, model selection, prompt logging policy, request shaping, and rollout control. Keep the boundary honest: the gateway should not hide runtime metrics, and runtime endpoints should still be observable directly during incidents.

Gateway API Inference Extension

The book's "API gateway to AI gateway" move becomes concrete in the Gateway API Inference Extension. The important API idea is InferencePool: instead of treating a Service as a flat set of pods, the gateway can treat an inference pool as a backend with an endpoint picker. The endpoint picker can look at inference-specific signals such as queue length, KV-cache utilization, active LoRA adapters, and model-server readiness before choosing a backend. This is the missing control loop in generic L7 routing.

The trade-off is a new failure mode: the endpoint picker itself is now on the request path or near it. Policies such as fail-open versus fail-close become product decisions. Fail-open protects availability but may route poorly during overload. Fail-close protects correctness and overload boundaries but can drop requests if the picker is unavailable. That is why the gateway belongs in observability and incident drills, not only in networking diagrams.

Mental Model

An LLM router is a scheduler above the scheduler. It chooses which warm model, cache state, adapter, and queue should absorb the next request.

Operational Checklist

  • Scale on token throughput, queue time, and GPU saturation rather than raw HTTP requests.
  • Route repeated prefixes to replicas with cache locality when the runtime can exploit it.
  • Keep LoRA/adapters warm and route by adapter residency to avoid loading on the hot path.
  • Use an AI gateway boundary for auth, rate limits, prompt logging policy, and model selection.

Failure Modes

  • HPA scales on CPU while the GPU queue is saturated.
  • Random load balancing destroys prefix-cache hit rate.
  • Scale-up creates cold replicas that accept traffic before model load completes.
