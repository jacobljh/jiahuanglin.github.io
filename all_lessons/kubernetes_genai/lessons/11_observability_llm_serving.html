Linear Story

Traditional observability asks whether the service is up, how many requests it handles, and how long they take. LLM observability adds token timing, queue timing, GPU state, model version, prompt shape, cache behavior, and quality. The book's observability chapter is where the platform starts to see the workload in its own language.

LogsRuntime events, model load, routing decisions, errors, policy actions.
MetricsTTFT, TPOT, tokens/sec, queue depth, GPU/HBM/DCGM, cache pressure.
TracesGateway, queue, prefill, decode stream, tools, guardrails, response.
QualityTask scores, hallucination checks, safety blocks, user feedback.

Metric Taxonomy

TTFT answers "how long before the user sees the first token?" TPOT answers "how steady is generation once it starts?" Throughput answers "how many tokens per second does the system produce?" Queue metrics answer "is the runtime saturated or under-routed?" GPU metrics answer "are expensive accelerators doing useful work?" Quality metrics answer "is the model still doing the right task?" These must be correlated by model, version, tenant, route, prompt length, and output length.

Privacy Trade-Off

Prompt logging is operationally tempting and legally sensitive. The platform needs sampling, redaction, access control, retention, and tenant policy. Observability that creates a data leak is not observability; it is a new incident source.

Mental Model

For LLMs, 'healthy' means the request path, token path, GPU path, and quality path are all inside bounds. One green HTTP status is not enough.

Operational Checklist

  • Emit request, queue, prefill, decode, and streaming spans with model and version attributes.
  • Track TTFT and TPOT separately and segment by prompt length and output length.
  • Correlate GPU utilization, HBM usage, cache pressure, and model-server queue metrics.
  • Keep prompt and completion logging governed by privacy policy, sampling, and retention rules.

Failure Modes

  • Only node-level GPU utilization is visible, so queueing and token latency remain mysterious.
  • Tracing drops streaming spans and hides where TTFT is spent.
  • Quality regressions ship because operational metrics stay green.
