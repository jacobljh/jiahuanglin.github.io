Learning in imagination

Real experience grounds the model; imagined experience multiplies it. Dyna and Dreamer exploit that asymmetry—but every cheap synthetic transition is only as trustworthy as the model region it visits.

The dependency
Lesson 07 established when an action-conditioned prediction can be interpreted as an intervention rather than a correlation. Now we ask what that model is for. Instead of using it only to predict, we turn it into a compact training environment. The linear chain is: collect real evidence → infer latent state → learn dynamics and outcomes → imagine action-conditioned futures → estimate returns → improve actor and critic → test the improved behavior in reality → repair the model where the policy exposes it.

1 · Why learn from a model at all?

Reinforcement learning begins with a costly asymmetry. Computation is cheap and repeatable; interaction may be slow, dangerous, or irreversible. A warehouse robot can run millions of matrix multiplications while one physical grasp still takes seconds and can damage an item. A model-free learner can reuse a real transition many times from replay, but it can update only from state–action consequences that were actually collected. It cannot cheaply ask what several unexecuted action sequences would have caused from the same grounded state. A world model tries to extract a reusable rule from many transitions, then applies that rule to candidate situations without paying the physical acquisition cost again.

Let a real transition be et = (ot, at, rt, ot+1, dt). Here ot is the observation, at the action, rt the reward after that action, and dt ∈ {0,1} an episode-ending indicator. A model-free update estimates a policy or value directly from stored et. A model-based learner additionally fits a transition distribution and outcome heads:

pθ(xt+1, rt, ct | xt, at)

Term by term: θ are learned model parameters; xt is the compact learned state or belief representation inferred from history; pθ is a predictive distribution rather than necessarily a single point; rt is feedback produced after action at; and ct is the corresponding continuation probability, usually near one for a continuing episode and zero at a true terminal. This is the same source-state → action → destination-state convention introduced in Lesson 04. Once the distribution is useful, sampling it yields imagined transitions. Those samples cost compute, not another collision.

The model does not create new evidence. It interpolates and recombines evidence already in the dataset. That sentence prevents a common conceptual mistake: ten thousand imagined steps are not equivalent to ten thousand independent real steps. They can reduce variance and expose an optimizer to more action sequences, but all inherit the same model assumptions and blind spots.

2 · Dyna is a schedule, not a particular neural network

Sutton’s Dyna architecture gives the simplest unifying picture. A single real transition supports three operations: improve the model, improve behavior from the real transition, and use the model for extra behavior updates. The original examples used tabular models and value learning, but the accounting survives with neural dynamics, actor–critic methods, or latent states.

real interaction → {direct RL update, model update} → K imagined RL updates

K is the imagination ratio: the number of model-generated updates per real step. At K = 0, Dyna reduces to a model-free learner. As K grows, one real experience is reused more aggressively. If the model is accurate where samples are drawn, learning can propagate reward information much faster. If it is wrong there, the same error is injected repeatedly.

Dyna-Q, one grounded outer step observe real (s, a, r, s′, done) store transition in replay update Q(s,a) toward r + γ max_a′ Q(s′,a′) # real update fit model M(s,a) ≈ (r,s′,done) # grounding repeat K times: # imagination inner loop sample an anchored state-action pair (ŝ, â) (r̂, ŝ′, d̂) ← M(ŝ, â) y ← r̂ if d̂ r̂ + γ max_a′ Q(ŝ′,a′) otherwise update Q(ŝ,â) toward y choose next real action; repeat

Notice what is and is not inside this inner loop. The model is ordinarily trained on real replay, not recursively on its own hallucinations. Imagined transitions train the decision system. Anchoring rollout starts in replay states prevents immediate departure from data support. Modern methods differ in how they represent state, produce actions, and calculate a target, but the same outer-loop/inner-loop separation is a useful debugging diagram.

3 · From pixels to a recurrent state-space model

Naively imagining future images is expensive. A 256 × 256 RGB frame has 196,608 intensity values, most describing appearance rather than decision-relevant change. Dreamer-style agents therefore learn a recurrent state-space model, usually called an RSSM. It combines a deterministic recurrent memory ht with a stochastic state zt. Memory carries predictable context; the stochastic variable represents information not determined by the past.

ht = fθ(ht−1, zt−1, at−1) posterior: qθ(zt | ht, ot)    prior: pθ(zt | ht)

Read the equations in execution order. First, recurrent function fθ advances deterministic memory using the previous memory, stochastic state, and action. When a real observation arrives, the posterior q combines that image with the predicted memory to infer zt. In imagination no future observation exists, so the agent samples the prior p using only ht. Training makes prior predictions agree with observation-informed posteriors without allowing the latent to collapse into an uninformative constant.

The full learned state given to task heads is xt = (ht, zt). From destination state xt+1, a decoder predicts ot+1, a reward head predicts rt, and a continuation head predicts ct. Their losses can be summarized as:

Lmodel = Lobs + Lreward + Lcontinue + β DKL[q(zt|ht,ot) ‖ p(zt|ht)]

Lobs makes latent state explain the sensor; Lreward preserves task signals that may occupy few pixels; Lcontinue models termination; the KL term aligns posterior and prior distributions; and β controls that pressure. Practical Dreamer variants use details such as KL balancing, free bits, symlog targets, categorical latents, and normalization. Those details change optimization behavior, not the basic division: the posterior grounds state in evidence; the prior is the simulator available between observations.

4 · Dreamer’s two loops, linearized

“Dreamer learns in imagination” is true but incomplete. There are two coupled loops with different data provenance. The world-model loop processes real sequences from replay and updates θ. The behavior loop starts from posterior states inferred along those real sequences, freezes or stops gradients into the model as appropriate, rolls the prior with actor actions, then updates actor and critic from predicted outcomes.

repeat: # Outer loop: acquire and ground act in the real environment with πφ; append observations to replay sample real sequence (o1:T, a1:T, r1:T, done1:T) infer posterior states x1:T with RSSM update world model θ using observation, reward, continuation, and KL losses # Inner loop: learn behavior in imagination choose posterior start states x0 from the real sequence for τ = 0 ... H−1: aτ ~ πφ(a | xτ) actor proposes intervention xτ+1 ~ pθ(x′ | xτ,aτ) prior advances latent world r̂τ ← Rθ(xτ+1); ĉτ ← Cθ(xτ+1) decode consequences of aτ v̂τ ← Vψ(xτ) critic estimates remaining return compute λ-return targets from r̂, ĉ, and Vψ update critic ψ toward stopped-gradient targets update actor φ to maximize imagined return plus exploration regularization

Start states matter. Beginning each imagination at an arbitrary prior sample asks the model to simulate states it may never have learned. Starting at replay posteriors gives a grounded root. Horizon H matters too: longer rollouts propagate effects farther but let transition errors compound. The actor is commonly optimized by backpropagating through differentiable latent dynamics, by a policy-gradient estimator, or by a mixture. The critic supplies a learned summary of consequences after the finite horizon.

Pixel decoding remains in the outer loop as a representation-learning signal, but no image needs to be rendered for every imagined step. That is the computational advantage. The actor consumes learned state x, not reconstructed RGB. If observation reconstruction is unnecessary or actively distracts from control, other self-supervised or task-aligned objectives can replace it, but then one must verify that the latent retains the physical factors future tasks need.

5 · Return is the bridge from prediction to behavior

An actor needs a scalar reason to prefer one imagined future. Define discounted return from time t as:

Gt = rt + γ ct rt+1 + γ² ctct+1 rt+2 + ···

γ ∈ [0,1) discounts distant outcomes; each ct gates terms after termination. A critic Vψ(xt) estimates expected future return under the current actor. Pure one-step bootstrapping has short model exposure but inherits critic bias. A full H-step imagined return uses more predicted rewards and less critic, but inherits more dynamics error. A λ-return mixes the alternatives:

Gtλ = r̂t + γ ĉt[(1−λ)Vψ(xt+1) + λ Gt+1λ]

Every symbol has an operational role. Hats mean model-predicted quantities. 1−λ is the weight on stopping the rollout target here and trusting the critic; λ is the weight on following the next predicted transition; the recursion ends at the imagination horizon with a terminal critic value. Small λ produces lower-variance, more bootstrapped targets. Large λ exposes learning to longer model rollouts. Neither setting removes bias; it chooses its source.

The critic minimizes a regression loss such as Lcritic = E[(Vψ(xt) − stopgrad(Gtλ))²]. The stop-gradient prevents the target from moving simply to match the critic. The actor maximizes an objective like J(φ) = E[Σ wt(Gtλ + η H[πφ(·|xt)])], where wt contains cumulative continuation and discount, H is policy entropy, and η rewards exploration. In continuous actions, constraints or squashing functions must keep actor outputs within the actuator’s valid range.

6 · Worked example: a robot approaches a fragile box

Consider a robot whose latent state encodes estimated distance to a box and whether the gripper is aligned. It imagines three steps. Its actor considers fast approach followed by grasp. The model predicts rewards r̂0 = 0, r̂1 = 2, and r̂2 = 6. It predicts continuation probabilities ĉ0 = 1.0, ĉ1 = 0.8, and ĉ2 = 1.0; the middle value falls because fast motion might knock the box over, while the final modeled transition is assumed to continue. Let γ = 0.9, λ = 0.5, and terminal critic estimate V(x3) = 4. Suppose V(x1) = 5 and V(x2) = 5.

Work backward. At the last imagined step, take G2λ = 6 + 0.9 × 1 × 4 = 9.6. At step one:

G1λ = 2 + 0.9 × 0.8 × [(1−0.5)×5 + 0.5×9.6] = 7.256

The bracket equals 2.5 + 4.8 = 7.3; multiplying by 0.72 gives 5.256; adding reward two yields 7.256. At the root:

G0λ = 0 + 0.9 × 1 × [0.5×5 + 0.5×7.256] = 5.5152

Now imagine a slow approach with lower immediate rewards but continuation near one, producing root return 5.9. The actor should prefer slow. This example reveals why the continuation head is not bookkeeping: it prices failure into every downstream reward.

Suppose, however, the training data contains only slow approaches. The reward model may extrapolate that even higher speed yields a perfect grasp while underestimating breakage. Gradient ascent can then push speed to the action limit and report return 12. The arithmetic is internally correct; the premises are false. A real trial breaks the box. The failure is model exploitation, not a λ-return bug. The fix requires support-aware policy updates, uncertainty, more grounded data, or a hard safety constraint—not merely a more accurate optimizer.

7 · Model exploitation is an optimization consequence

Validation error averages over a dataset distribution. Policy optimization induces a different distribution: it searches for actions with maximum predicted return. If prediction error is ε(s,a) = Q̂(s,a) − Q(s,a), maximizing Q̂ = Q + ε favors both genuinely good actions and actions with positive error. Even zero-mean error becomes positively selected after a maximum. The actor is therefore an adaptive adversary against its own simulator.

Three failure modes should be separated. Compounding error occurs when small transition mistakes move later rollouts into unfamiliar state. Reward hacking occurs when the actor finds a latent feature that a reward head mistakes for success. Support extrapolation occurs when proposed state–action pairs were absent from real data. They often co-occur, but diagnostics and mitigations differ.

A naive fix is to pick one regularizer and call the imagination loop safe. That fails because the error can enter at different points: the rollout may start ungrounded, drift with depth, cross action support, exploit an uncertain reward, violate a known rule, or remain uncorrected after deployment. Derive defenses in that same order:

  1. Ground the root with replay. Replay-rooted starts begin from posteriors that actually absorbed observations. This removes arbitrary fantasy at step zero, but it may not train recovery from rare off-policy states. Diagnose the boundary by comparing error from replay roots with error from perturbed roots.
  2. Limit how long an unchecked prediction chain runs. A short imagined horizon bootstraps before dynamics drift far. The cost is transferred trust: terminating simulation early places more weight on critic accuracy. Sweep horizon while logging transition error and terminal-value contribution so the source of bias is visible.
  3. Ask several plausible models where knowledge ends. Ensemble disagreement can penalize actions for which models fitted to the evidence predict different consequences. This costs extra compute, and shared architecture or data bias can fool every member in the same way. Validate disagreement against actual error rather than treating diversity as calibrated uncertainty automatically.
  4. Keep the actor near actions whose consequences were observed. Behavior regularization limits divergence from demonstrated or replay actions, directly addressing support extrapolation. Its price is conservatism: a strong penalty can prevent genuine improvement beyond the behavior policy. Plot achieved improvement against distance from behavior instead of selecting the penalty from offline return alone.
  5. Price uncertainty inside the return. An uncertainty-aware objective optimizes predicted return minus a risk coefficient times uncertainty. This makes an apparently brilliant but poorly supported future less attractive. Calibration is difficult, and excessive pessimism can stall useful exploration, so test whether predicted risk ranks real errors and tune the coefficient to consequence severity.
  6. Refuse to trade known safety rules for learned reward. A hard constraint or shield rejects actions that violate trusted geometry, torque, or safety limits even if the reward head is optimistic. The constraint itself must be reliable, and it may reduce reachable behavior. Stress-test the shield separately because a learned planner cannot compensate for a mistaken safety boundary.
  7. Let reality answer the new policy’s questions. Online correction collects evidence exactly where the improved policy visits, turning exploitation failures into targeted model updates. Real interaction has a cost and needs safe deployment gates, so begin in shadow mode or with bounded action changes and expand only after imagined-return calibration holds.

This sequence exposes the implementation consequence of the physical problem. Because synthetic experience contains no new evidence, an imagination system needs a grounded root, a bounded prediction interval, a measure of ignorance, a policy-support rule, independent constraints, and a controlled path back to reality. Removing any one link creates a recognizable diagnostic signature rather than a mysterious drop in final return.

A practical pessimistic score is Jsafe = E[Ĝ] − κ σensemble(Ĝ). The coefficient κ converts epistemic uncertainty into a return penalty. This is not a universal safety guarantee: ensemble disagreement measures only variation inside the chosen model family. Known constraints should remain explicit.

8 · Diagnostics that localize the failing loop

Do not begin with total episodic return alone. It is the final symptom of a coupled model, critic, and actor. Instrument each boundary:

  1. Posterior-versus-prior error by horizon. Start from a real posterior, roll the prior under recorded actions, and compare predicted rewards, continuations, features, and decoded observations with the held-out trajectory. Plot error at each step, not just an average.
  2. Action sensitivity. Hold the start state fixed and perturb actions. If predicted states and rewards barely change, the model ignores control. If infinitesimal changes cause huge rewards, gradients are exploitable.
  3. Predicted-return calibration. Bucket real executions by imagined return and compare prediction with achieved return. Optimistic top buckets are especially dangerous because policy improvement selects them.
  4. Policy support. Measure actor action likelihood under replay behavior, nearest-neighbor distance in latent state–action space, or ensemble disagreement. Track these through an imagined rollout.
  5. Critic target decomposition. Log immediate predicted rewards, continuation products, and terminal bootstrap separately. A high value caused entirely by a terminal critic requires a different fix from a spurious reward spike.
  6. Real-versus-imagined ablation. Compare the same learner with K = 0, short imagination, and long imagination. If more model usage monotonically hurts, inspect model bias before tuning the actor.
  7. Counterfactual rollouts. From matched real roots, test safe action sweeps against known monotonicity, conservation, and collision constraints. This directly probes what the actor will optimize.

Also watch representation collapse, KL spikes, reward scale, gradient norms through time, continuation imbalance, and actor entropy. A binary continuation head trained on data with almost no terminals can predict “continue forever,” making imagined rewards effectively immortal. Balanced sampling or calibrated losses may be required.

9 · Online and offline imagination have different contracts

In online learning, a newly improved policy eventually visits the states it imagines. Surprising outcomes enter replay, the posterior grounds them, and the model can repair policy-relevant errors. This creates a corrective cycle, although unsafe proposals still need deployment gates. In strictly offline learning, the fixed dataset never answers the actor’s new questions. The inner loop can repeatedly optimize beyond support with no automatic reality check. A high imagined return is then a hypothesis that the dataset may be unable to identify.

That difference changes algorithm design. Online systems can alternate conservative improvement with small real-data expansions. Offline systems typically need stronger behavior regularization, pessimistic values, support-aware action constraints, or uncertainty penalties. One useful objective is Joffline(φ) = E[Ĝ] − α D(πφ(·|s), πb(·|s)). Here πb approximates behavior in the dataset, D measures divergence, and α prices extrapolation. Large α stays close to demonstrated action; small α permits improvement but asks more of the model.

Do not describe an offline world model as “safe because it never explores.” The real environment is safe during training precisely because it is absent; deployment risk can be higher if simulated optimization has exploited untested regions. Before deployment, use held-out policies where available, a trusted simulator, constraint stress tests, shadow mode, and bounded real trials.

10 · Choosing the imagination budget as a system designer

The right horizon, batch size, and update ratio depend on the bottleneck. If environment steps dominate cost and dynamics are smooth, spend more compute per real transition. If the world changes faster than the replay buffer, excessive inner-loop optimization makes the learner stale. If latency matters only during deployment, heavy offline imagination can amortize into a fast actor. If goals change online, a single amortized policy may be less flexible than the planner in Lesson 09.

Measure three currencies separately: environment steps, accelerator operations, and wall-clock time. “Sample efficient” can conceal a hundred-fold compute increase. For a robot fleet, data diversity across machines can be worth more than additional gradient steps on one machine’s narrow replay. For an offline dataset, no online correction is available, so conservative support constraints become central. For safety-critical control, an actor should be deployed behind uncertainty thresholds, rate limits, a fallback controller, and shadow evaluation before its actions reach hardware.

A useful staged recipe is: first make one-step reward and continuation calibrated; then verify open-loop latent rollouts under recorded actions; then train a critic with the actor frozen; then enable short-horizon actor learning under action constraints; then compare imagined and real return rankings; finally expand horizon or imagination ratio only when those checks remain stable. This linear staging sacrifices some experimental speed but makes causality in debugging much clearer.

11 · Dreamer, Dyna, and planning are related but not identical

Dyna describes the architectural idea of interleaving real learning, model learning, and simulated updates. Dreamer is a particular modern realization: it learns a latent RSSM and trains an actor–critic on multi-step imagined latent trajectories. Its actor amortizes optimization: after training, one forward pass proposes an action. Model predictive control instead performs action optimization at decision time. These can be hybridized. A learned actor can propose sequences for MPC; a planner can provide targets for policy distillation; real planner failures can enrich replay.

The central trade is where computation lives. Actor learning pays optimization cost during training and gives fast deployment, but compresses behavior for its training task distribution. Online planning pays repeatedly at deployment, but adapts naturally to a current goal, constraint, or observation. Both remain bounded by model validity.

Where this points next

Imagination has now been used to train a reusable decision rule. The alternative is to keep the current belief and goal online, generate candidate action sequences at decision time, and spend computation on the particular state in front of us. Lesson 09 derives that branch: random shooting and CEM for continuous trajectories, MPC for corrective replanning, MCTS for discrete branching, and MuZero for learning only the latent information search needs.

Takeaway
Imagined experience is cheap computation built from expensive evidence, not new evidence. Dyna names the real/model/simulated update schedule; Dreamer grounds an RSSM with real observations and trains actor and critic through prior latent rollouts. λ-returns choose how much to trust predicted rewards versus a value bootstrap. Because optimization selects optimistic model errors, successful systems root imagination in replay, limit or penalize unsupported rollouts, instrument return calibration, and repeatedly close the loop with reality.

Interview prompts

  • What is the exact difference between Dyna and ordinary experience replay? Replay reuses transitions that actually occurred. Dyna additionally fits a model and generates transitions that did not occur, then uses them for decision-learning updates. Its benefit and unique risk both come from that synthetic branch.
  • Why does Dreamer need both posterior and prior latent distributions? The posterior uses the current observation to ground latent state during real training. The prior predicts state without a future observation, so it is the distribution available inside imagination. Their alignment teaches the prior to anticipate posterior states.
  • Why can Dreamer learn behavior without decoding pixels inside every rollout? Reward, continuation, critic, and actor operate directly on the learned latent state. Pixel reconstruction can shape that state in the model-training loop, while imagined policy learning uses compact dynamics and task heads.
  • What does λ control in a λ-return? It interpolates between earlier value bootstrapping and following more imagined rewards before bootstrapping. Larger λ depends more on multi-step model accuracy; smaller λ depends more on critic accuracy.
  • Why is a policy an adversarial consumer of a learned model? It maximizes predicted return rather than sampling average data. Maximization preferentially selects actions with positive prediction error, so even unbiased validation error can become optimistic under the optimized policy.
  • How would you tell reward-model exploitation from transition drift? Decompose predicted return and compare action-conditioned rollouts with held-out reality. An isolated reward spike in an otherwise plausible latent path implicates the reward head; growing state and outcome error with horizon implicates dynamics drift.
  • Why do short rollouts help, and what new dependency do they create? They keep imagined states closer to grounded replay and limit compounding error. They terminate earlier with a critic bootstrap, so behavior becomes more sensitive to critic bias.
  • When would online planning be preferable to a Dreamer actor? When goals or constraints change frequently, extra decision-time compute is available, and adapting search to the current belief matters more than a one-pass action. A hybrid can use the actor to seed the planner.
