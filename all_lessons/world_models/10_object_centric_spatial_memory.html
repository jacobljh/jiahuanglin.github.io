Object-centric spatial memory

Derive the representation from the questions an agent must answer: what exists, where it is, which observation belongs to it, and what remains true when it cannot be seen.

The dependency
Earlier lessons compressed history into a belief and used that belief for prediction and planning. This lesson asks what should be inside that belief when the world contains persistent things. The linear path is: downstream query → persistent entity → binding → occlusion update → relation dynamics → training and diagnosis.

1 · Begin with a query, not with a fashionable representation

Imagine a mobile robot clearing breakfast from a kitchen. At time t=0, it sees a blue mug on the counter, a red bowl beside it, and a closed cupboard. It turns toward the sink. The mug leaves the camera view. A person then walks between the robot and the counter. Five seconds later, the robot must answer: “where is the mug, is the path to it free, and what will happen if I reach from the left?” A single embedding of the latest frame may contain rich appearance, yet the required answers concern an entity that is currently absent from that frame.

First principles starts from the computation. A representation is useful when it makes frequently needed queries cheap and reliable. The kitchen robot needs at least four kinds:

  • Existence and identity: is there one mug or two, and is the reappearing blue patch the same mug?
  • Metric space: where is it relative to the robot, counter edge, and free space?
  • Interaction: what supports it, what blocks it, and which objects would move if pushed?
  • Persistence under missing evidence: what should remain in memory while a surface is outside view or occluded?

A useful structured belief therefore separates entities from the current image:

bt = {et1, …, etNt, Mt, Rt, gt},    eti = (ιti, pti, vti, fti, uti)

Here ιi is a persistent identity hypothesis, pi,vi are pose and velocity, fi stores appearance or semantic features, and ui stores uncertainty. M is spatial memory such as occupancy or free space, R contains relations, and g is global context. This is a belief, not a perfect inventory: every component may be uncertain, and Nt itself may be unknown.

2 · The running example exposes why persistence is necessary

Follow the blue mug through a minimal filter. When it is first detected, the model creates a track with an estimated world position and covariance. While visible, visual evidence corrects that estimate. When the camera turns away, the model does not delete the mug. It predicts the entity forward, perhaps with a nearly-static dynamics model. Uncertainty grows because a person could have moved it. When the mug reappears, the model compares the new observation against all surviving hypotheses and either updates the old track or creates a new one.

ēti = Te(et−1i, at−1, mt−1i)   [predict even when unseen]
eti = U(ēti, otj)   [correct if observation j is assigned to i]

The message mi summarizes relevant interactions: a hand contacting the mug should change its predicted motion; a distant spoon usually should not. The update U is conditional on an assignment between observation j and entity i. Thus object permanence is not a slogan. Computationally, it is the decision to keep predicting an entity belief when the observation likelihood is absent, while widening uncertainty rather than pretending certainty.

3 · Choose representation by query cost

No state format is universally best. “More explicit” is not automatically better; it may make updates brittle or expensive. Estimate how often a query is asked, how costly it is to answer, and how damaging approximation is. For the kitchen robot, collision checking runs thousands of times during planning, so a spatial grid can be valuable even if it duplicates information stored in object tracks. Finding “the blue mug” is naturally an entity query. Producing a novel camera view is naturally dense.

To see why different representations coexist, derive each one from the first query that becomes painfully expensive:

  1. Start with the entity question. The robot asks, “how many things are present, which one is the mug, and how will that particular thing move?” If the state is only a dense image-like field, answering requires searching or segmenting many locations every time. That failed lookup suggests a named, indexable unit. Object slots or tokens make counting, attribute retrieval, and per-entity prediction roughly O(N). The implementation consequence is a set of entity records. The price is that the model must decide where one entity ends and another begins, so merge, split, and identity-swap errors become the diagnostic targets. Slots also remain awkward for pixel-perfect rendering and amorphous background.
  2. Now add time. A fresh object set for each frame can answer “what is here now,” but it cannot directly answer “where was this mug going, or how long has it been unseen?” Recomputing history from raw frames is too slow and can assign a new identity after every occlusion. That failure derives the need for tracking memory: keep trajectory, velocity, time since seen, and a direct track-ID lookup. The consequence is efficient temporal queries, but also dependence on the detector that initialized and updated each track. If detection is wrong, memory inherits the error. Unknown objects and complete free-space coverage still need another representation.
  3. Then ask an interaction question. Suppose the robot needs to know what supports, contains, or touches the mug. Testing every possible geometric relation from scratch wastes computation and may be ambiguous under occlusion. This motivates a scene graph, where the query becomes a local traversal around the mug node. The implementation consequence is an explicit relation vocabulary and edge-update mechanism. The corresponding diagnostic is stale or impossible edges; the deeper risk is a brittle ontology that cannot express an unanticipated relation or continuous geometry.
  4. Then ask the control loop’s repeated safety question. A motion planner may check thousands of points for free space, visibility, or collision. Performing an entity search for every point is the wrong computational shape. A bird’s-eye-view or occupancy grid turns these checks into local array accesses. The consequence is fast spatial planning, but position is quantized into cells. Thin geometry can disappear, vertical structure can collapse, moving objects can smear across updates, and identity is not native. Cell size, cleared trails, and collision misses are therefore the right diagnostics.
  5. Finally ask for an observation. The robot may need to reconstruct appearance or predict what a sensor would see at an arbitrary location. A short list of object attributes usually cannot reproduce texture, lighting, and everything that did not receive an object label. A dense feature map or neural field makes local appearance and sensor reconstruction native. The consequence is a rich residual representation, but retrieving “the mug” may again require search or segmentation over many samples. Identity and causal variables can remain implicit even while rendered images look excellent.

The practical answer is usually hybrid. Keep a metric map for navigation, persistent object tracks for manipulable things, a graph for sparse interactions, and dense residual features for what the ontology misses. Duplication is acceptable if synchronization is explicit: after the mug moves, both its entity pose and occupied cells must update. The representation should be judged by total system cost—encoding, update, queries, and inconsistency—not merely latent size.

Linear choice rule
Begin with the physical query, attempt it using the simplest current state, identify the operation that becomes unreliable or expensive, and add only the structure that makes that operation direct. Then name the new failure mode the structure introduces. Representation design is this loop repeated—not a contest to find one universal latent.

4 · Object discovery creates hypotheses, not ground truth

Where do entities come from? A supervised system can initialize them from detections and masks. An unsupervised slot model can let a fixed set of vectors compete to explain image regions. A query-based transformer can decode a variable set of object hypotheses plus a “no object” outcome. In all cases the decomposition is an inference choice.

Slot attention illustrates the mechanism. Each image feature sends evidence to several slot queries; normalized attention makes slots compete; an iterative update aggregates assigned evidence. A decoder may render a component and mask from each slot, then compose them. Reconstruction pressure encourages coverage, while competition encourages separation. But reconstruction alone does not say that the mug should occupy the same slot tomorrow. A model can permute slots on every frame and reproduce all pixels equally well.

Object is a useful hypothesis
Water, smoke, shadows, reflections, cloth, articulated bodies, crowds, and “the open space between chairs” do not admit one canonical object decomposition. Choose entities whose state predicts interventions and answers downstream queries. Preserve a dense residual channel so the world is not forced into an incomplete list of nouns.

5 · Binding is a constrained inference problem

Suppose a second blue mug crosses behind the first. At the next frame there are two detections and two predicted tracks. Binding asks which measurement updates which track. A common cost combines predicted position, appearance, shape, class, and motion consistency:

Cij = λpd(p̂ti, ptj) + λfd(f̂ti, ftj) + λsd(ŝti, stj) − λv log P(vtj | vt−1i)

A one-to-one matcher minimizes total cost, augmented with “new entity” and “missed observation” assignments. Hard matching yields clear identities but can make one irreversible error. Soft attention keeps alternatives differentiable but may blend two mugs into an average entity. A stronger belief retains multiple association hypotheses until evidence resolves them, at higher memory and compute cost.

Identity is not appearance. Lighting changes; an object rotates; identical manufactured objects really are indistinguishable from one view. Identity is continuity of a physical hypothesis through space, time, and interaction. If mug A is picked up while mug B remains supported by the counter, contact supplies stronger identity evidence than color. This is why temporal dynamics and relations should participate in matching rather than being applied only after matching.

Slot swapping occurs when two latent indices exchange physical referents. Per-frame reconstruction barely notices, but a learned dynamics head now applies A’s velocity, ownership, or affordance to B. Diagnose it with identity survival through crossings and long occlusions, not just adjacent-frame association accuracy.

6 · Occlusion is negative evidence only when visibility was expected

A missing detection has several causes: the mug left the field of view, another object occluded it, the detector failed, or the mug was removed. Treating all absence as nonexistence destroys permanence; treating all absence as harmless creates immortal ghost objects.

The model therefore predicts visibility from geometry. Let Vti mean that entity i should be visible from the current sensor pose. The observation update distinguishes cases:

  • Not in view: preserve the entity and evolve it from dynamics; absence carries almost no information.
  • Predicted occluded: preserve it, grow pose and attribute uncertainty, and maintain the occluder relation.
  • Predicted visible but not detected: decrease existence probability because absence is informative.
  • Reobserved: bind, correct the state, and measure whether uncertainty was calibrated.
P(Eti | ¬Dti, Vti) ∝ P(¬Dti | Eti, Vti)P(Eti),    P(¬D | E, V=0) ≫ P(¬D | E, V=1)

In plain language: not seeing the mug matters only if the model believed it had a clear chance to see it. Track deletion should use this visibility-aware existence belief, not a fixed number of missed frames. For safety, a planner may keep a low-probability obstacle longer than a low-probability decorative object because the cost of being wrong differs.

7 · Relations turn global dynamics into local computation

The mug’s next state depends mainly on nearby supports and contacts. Build a time-varying graph with entities as nodes and candidate interactions as edges. A graph dynamics step sends messages and aggregates them in a permutation-invariant way:

mij = φe(eti, etj, rtij, at),    et+1i = φv(eti, Σj∈𝒩(i)mij, ati)

This factorization reuses the same rule for any object count. If the robot pushes the bowl rather than the mug, the contact mechanism can transfer. Sparse neighborhoods reduce computation from all-pairs O(N2) to roughly O(|E|). But edges are themselves beliefs. Contact begins and ends; support can transfer from counter to gripper; containment may be hidden. Predict relation probabilities and let geometry prune impossible edges without assuming the graph is fixed.

Relations also protect binding. If a hand and mug move together after contact, their coupled trajectory is evidence. Conversely, an impossible teleport or a sudden support change may signal an identity swap rather than exotic physics.

8 · Training must reward persistence and usefulness

A reconstruction loss alone allows temporally inconsistent decompositions. Training signals should target the behaviors the memory exists to support:

  • Observation reconstruction: decoded entities and background explain visible RGB, depth, masks, or flow.
  • Temporal consistency: matched entities predict future pose and appearance; contrastive objectives pull the same track together and separate others.
  • Set supervision: permutation-invariant matching compares predicted and labeled entity sets without assigning meaning to slot index.
  • Occlusion curriculum: mask objects or train on long natural occlusions, then score identity and calibrated existence when they return.
  • Dynamics and relation loss: predict actions’ effects, contact, support changes, and future occupancy.
  • Task loss: train or evaluate planning queries—collision, retrieval, manipulation success—so beautiful decomposition is not the only objective.

A common joint objective is a weighted sum, L = Lobs + αLset + βLdyn + γLid + δLtask. Weighting matters. Excessive per-pixel pressure can devote slots to texture; excessive identity regularization can resist legitimate split/merge events such as opening an articulated object. Staged training often helps: learn perception, introduce temporal matching, then fine-tune dynamics and task heads while continuing to test the whole loop.

9 · Cardinality and lifecycle are part of the state

So far the set of entities looked fixed, but real worlds contain births, exits, assembly, separation, and changes in relevance. A person can carry a new plate into the kitchen; the robot can pour several ice cubes from a tray; a cupboard can open and reveal objects that existed before they were observed. The system must distinguish new to the world from new to the sensor. Usually it cannot know immediately, so it should represent a tentative birth rather than force a binary decision.

A practical lifecycle has stages. An unmatched detection begins as a candidate with low existence confidence. Repeated compatible evidence promotes it to an active track. Missed evidence sends it into an occluded or dormant state rather than deleting it immediately. A track can later be reactivated, archived, merged with a duplicate, or terminated when visible absence and exit geometry make disappearance likely. Thresholds should depend on detection reliability and task risk.

ρti = P(Eti=1 | o1:t, a1:t−1),    ρti ↓ when expected visible but missed;    ρti roughly persists when occluded or out of view

Variable cardinality complicates neural architectures. A fixed number of slots provides efficient batching, but “empty” slots need an explicit presence variable, and crowded scenes exceed capacity. A large fixed budget wastes compute and encourages duplicate slots. Dynamic query sets match cardinality more naturally but make memory allocation and accelerator batching harder. Some systems reserve a moderate active set, summarize distant objects into a map, and retrieve or reactivate tracks when the camera returns.

Split and merge events require semantic care. Two image regions that separate may be two physical objects revealed from occlusion, two parts of one articulated entity, or one object that physically broke. Conversely, adjacent components may be a stack, a temporary contact pair, or one object. Rather than make slot count carry all meaning, represent hierarchical entities and relations. The cupboard is an entity; its door and body are persistent parts connected by a joint. The mug and saucer are separate entities linked by support. The appropriate level can then change by query: navigation treats the stack as one obstacle, manipulation keeps two identities.

Background also needs memory. Walls, counters, free space, lighting, and shadows should not consume the same per-object machinery as movable mugs, yet calling everything “background” can hide obstacles and manipulable unknowns. A useful decomposition has a static or slowly changing spatial layer, active dynamic entities, and a residual appearance field. When an unexplained region begins moving or becomes task-relevant, the model can promote it from residual to entity. When a track remains static for a long time, it may be fused into the map while retaining provenance so later motion can undo that fusion.

10 · Evaluate the exact invariances memory promises

A single detection metric does not measure persistent world belief. Build evaluation sequences around controlled transformations. For viewpoint invariance, move the camera around a static kitchen and verify that world-frame entities stay fixed while visibility changes. For identity, cross visually identical objects, fully occlude one, and score which history each reappearing object inherits. For existence calibration, group predictions by confidence and check whether objects actually survive at the advertised rate. For interaction, intervene on one member of a relation and measure whether only causally connected entities respond.

Also evaluate queries at their native operating point. Report collision false negatives for the BEV at the robot’s safety margin, retrieval latency as object count grows, position error versus occlusion duration, and planning success using the stored state. If a representation saves memory by quantizing space, plot downstream failures against cell size. This reveals the product tradeoff far better than reporting an average embedding loss.

Counterfactual corruption tests are especially diagnostic. Replace appearance while preserving trajectory: does identity follow continuity or color? Perturb camera pose while keeping pixels: does the map expose an inconsistent transform? Delete the observation of a visible mug: does existence fall more than when a wall occludes it? These interventions test the inference rule the model claims to implement.

Set metrics need careful matching. Detection precision can count a duplicate track as only one extra false positive, while the planner sees two obstacles and the dynamics predicts two collisions. Report duplicate rate, fragmentation (one physical object becoming several track IDs), merge rate (several objects becoming one), identity switches, and time to reacquire after occlusion. Stratify results by occlusion length, visual similarity, speed, crowding, and distance. An aggregate score otherwise rewards easy visible objects and conceals exactly the persistence problem the representation was introduced to solve.

Finally, test compositional generalization. Train interactions with two or three objects, then evaluate more objects, novel arrangements, and familiar relations attached to unfamiliar appearances. Shared entity and graph updates should degrade gradually. If performance collapses merely because the mug is green or an extra spoon is present, the state has names that look compositional without actually learning reusable dynamics.

11 · Diagnose the mechanism, not only the final score

A final score says that some downstream behavior failed, but it does not say which update rule to change. Diagnose linearly: hold the scene fixed, vary one source of evidence, observe the first internal state that becomes wrong, and repair that mechanism before retraining everything.

  1. The mug vanishes whenever the robot turns. The physical mug did not disappear; only the camera lost access to it. A frame-only state, or a deletion rule that ignores predicted visibility, confuses “not measured” with “does not exist.” Hold out increasingly long out-of-view intervals and inspect the track before the mug returns. The repair is a world-frame track that survives while uncertainty grows. The diagnostic passes only if reobservation corrects that same hypothesis rather than creating a new mug.
  2. Two blue mugs exchange trajectories. The naive matcher trusted similar appearance even when the physical paths and contacts disagreed. Recreate a crossing while controlling color, motion, and hand contact. If identity follows color rather than continuity, binding is appearance-dominated. Add motion and relation evidence, or retain multiple association hypotheses until the crossing resolves, then measure identity survival rather than adjacent-frame overlap.
  3. A ghost mug blocks the planner forever. Persistence solved premature deletion, but the naive repair made existence immortal. Put the predicted mug in a clear, visible region and repeatedly fail to detect it. This absence should reduce survival probability much more than absence behind a wall. Make expected visibility negative evidence and calibrate the resulting existence probability against actual survival.
  4. The occupancy map leaves a smeared trail. The system fused every observed occupied cell into a supposedly static layer. When the mug moved, its old cells remained occupied. Replay one controlled translation and inspect map layers rather than only the final path. Separate static structure from moving entity occupancy, update the dynamic layer from the track, and explicitly clear vacated cells.
  5. Rendering looks convincing, but grasping fails. The training objective rewarded pixels, so the representation learned texture without the metric pose, support, or contact precision the controller needs. Hold appearance quality constant and probe those variables directly. Add explicit geometry or a task loss at the failing spatial scale, then verify actual grasp success rather than assuming that better images imply better control.
  6. Memory cost grows without bound. The model can create and preserve tracks but has no lifecycle policy for duplicates, old evidence, or low-value entities. Drive through a long sequence and plot active state by age, confidence, and task relevance. Merge duplicate hypotheses, archive low-value tracks, and retain uncertainty summaries according to decision risk. The test is graceful bounded growth without forgetting safety-critical entities.

12 · Product and system tradeoffs

Structured memory creates operational responsibilities. Variable entity counts complicate batching. Association adds latency. A high-resolution BEV makes collision queries cheap but consumes memory over large areas. Keeping raw keyframes supports later re-identification but raises bandwidth and privacy costs. Learned slots reduce labeling but are harder to debug than detector-backed tracks.

Choose a tiered memory. Run fast local occupancy and active tracks at control frequency; update dense appearance or a global map more slowly; archive uncertain historical evidence for recovery rather than placing it in every planning step. Attach timestamps, confidence, provenance, and coordinate frames to each fact. When sensors disagree, provenance lets the system retract an update instead of silently averaging incompatible states.

Finally, tune lifecycle thresholds against decision cost. A home robot may conservatively keep a possible child or pet as an obstacle. It can aggressively forget a spoon’s color. “What to remember” is therefore a product policy as well as a representation choice.

The update interface should make uncertainty and timing unavoidable. Instead of returning only get_pose(id), return a pose distribution, timestamp, visibility state, and source. A planner can then enlarge a collision envelope for an old track, request a new view before a delicate grasp, or reject an entity inferred from an untrusted sensor. This prevents downstream code from converting a probabilistic memory into accidental certainty.

Privacy and retention also influence design. Persistent appearance embeddings can re-identify people; keyframes can contain sensitive detail. If the task only needs occupancy and coarse motion, discard identity-rich features after association or keep them on-device for a limited interval. Entity memory makes selective deletion easier than an opaque history embedding: the system can remove one person’s appearance while preserving anonymous free-space evidence. But duplicated state means deletion must propagate to maps, tracks, retrieval stores, and logs.

Measure operational behavior under load. When object count spikes, does association miss its deadline, silently drop low-confidence tracks, or degrade all tracks? A deterministic resource policy should prioritize safety-critical classes and nearby interactions, summarize low-value distant entities, and expose overload to the planner. Graceful degradation is part of representation quality: a theoretically elegant all-pairs model that misses control deadlines is not useful state.

13 · Bridge: objects need a stable place to live

The object-centric belief solves one problem: the mug persists as an entity instead of being rediscovered from every frame. It exposes the next problem: coordinates. If its position is stored only in camera pixels, turning the camera makes every stationary entity appear to move. Lesson 11 separates sensor pose from world state and asks when explicit 3D/4D geometry makes prediction and planning queries cheaper.

Takeaway
Object-centric spatial memory is a visibility-aware belief over persistent entities, relations, and free space. Derive it from downstream query cost; treat assignment and existence as uncertain inference; train through occlusion and intervention; and keep a dense residual for aspects that do not fit clean object boundaries.

Interview prompts

  • What is object permanence computationally? It is predicting an entity belief when its measurement is absent, preserving identity while increasing uncertainty, then correcting that same hypothesis when evidence returns.
  • Why is object discovery not enough for tracking? A per-frame decomposition is permutation-invariant. Tracking additionally binds current observations to persistent physical hypotheses using motion, appearance, geometry, and relations.
  • When should absence delete an object? Absence is strong evidence only when the object was predicted to be visible and the detector had a good chance to see it. Out-of-view or geometrically occluded absence should mainly widen uncertainty.
  • How would you choose slots versus BEV? By query cost: slots make count, attributes, and per-object dynamics cheap; BEV makes free-space and collision queries cheap. Many systems need both.
  • What is slot swapping, and why can reconstruction miss it? Two latent indices exchange physical identities. The same components can still reconstruct both objects, but their histories, velocities, and affordances become attached to the wrong thing.
  • Why use graph dynamics? Most interactions are local. Shared message functions provide compositionality and permutation symmetry, while sparse edges avoid all-pairs computation.
  • When is an object decomposition a harmful bias? When dynamics depend on amorphous fields, material continua, crowds, or task-defined regions without stable boundaries. Hybrid entity-plus-dense states reduce the mismatch.
  • How would you evaluate memory beyond detection accuracy? Test identity through crossings and long occlusions, existence calibration, relation transitions, map consistency, query latency, and whether plans based on the memory succeed.
